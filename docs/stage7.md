## Day 61: 参数记忆

**一句话**：模型知识被编码在神经网络权重中，通过反向传播调整存储。

### 核心定义

- **参数记忆**：知识存储在 transformer 层的权重矩阵中
- **存储方式**：不是显式数据库，而是统计模式关联
- **形成过程**：数十亿次反向传播，调整权重矩阵
- **局限性**：静态（截止训练日期）、可能产生幻觉

### 关键公式

<div class="formula-box">

```
预训练流程：
输入文本 → Embedding → Transformer 层 → 预测下一个词
                    ↓
              计算损失 L = -log P(正确词)
                    ↓
              反向传播：∂L/∂W
                    ↓
              更新权重：W = W - η∂L/∂W
```

</div>

### AI 中的应用

- 知识编辑：直接修改特定权重注入知识
- 模型融合：合并多个模型的权重
- 持续学习：在不遗忘旧知识的情况下学习新知识

## Day 62: 上下文记忆

**一句话**：模型通过提示词中的上下文临时"记住"信息，无需更新权重。

### 核心定义

- **上下文记忆**：通过 Prompt 中的上下文临时学习
- **特点**：无需微调、即时生效、受限于上下文窗口
- **窗口长度**：通常 4K-128K tokens
- **机制**：Attention 让模型"看到"所有上下文信息

### 关键公式

<div class="formula-box">

```
In-Context Learning 流程：
Prompt: "猫→cat, 狗→dog, 房子→house, 车→？"
        ↓
每个 token 计算 Attention：
Query: "车"的 Query 向量
Key: 所有上下文的 Key 向量
Value: 所有上下文的 Value 向量
        ↓
Attention 分数 = softmax(Q·K/√d)
        ↓
聚合 Value：输出 = Σ(Attention 分数 × Value)
        ↓
预测："车" → "car"
```

</div>

## Day 63: Q-K-V 机制详解

**一句话**：Q-K-V 通过相关性筛选，让模型"点亮"相关知识区域。

### 核心定义

- **Query (Q)**：我要找什么信息？
- **Key (K)**：我有什么信息？
- **Value (V)**：信息的具体内容是什么？
- **注意力分数**：Q·K（点积），度量相关性

### 为什么能"点亮"知识

<div class="formula-box">

```
"首都"的 Query 向量：
- 与"北京"的 Key 点积 → 分数很高 ✓ (激活)
- 与"政治职能"的 Key 点积 → 分数较高 ✓ (激活)
- 与"烤鸭"的 Key 点积 → 分数很低 ✗ (抑制)

结果：
"首都"主要聚合了"北京"、"政治"等信息
忽略了不相关的"烤鸭"、"旅游"等信息
```

</div>

### Softmax 放大效应

<div class="formula-box">

```
原始分数：[8, 7, 2, 1]
Softmax 后：[0.46, 0.34, 0.12, 0.08]
→ 前两项占 80%

原始分数：[20, 2, 1, 1]
Softmax 后：[0.999, 0.0004, ...]
→ 第一项占 99.9%（极端聚焦）
```

</div>

## Day 64: Prompt 工程原理

**一句话**：Prompt 通过注意力机制引导模型在概率模式中找到正确路径。

### 三种影响机制

| 机制 | 说明 | 示例 |
|------|------|------|
| **激活知识区域** | 通过注意力"点亮"相关区域 | "政治职能"→激活政治相关知识 |
| **设定概率偏向** | 改变后续 token 的概率分布 | "深思熟虑"→更长推理链 |
| **引导输出格式** | 指定回答的结构和形式 | "列举 3 点"→结构化输出 |

### Prompt 优劣对比

| Prompt 类型 | 效果 | 注意力机制 |
|------------|------|-----------|
| 模糊 Prompt | 激活混杂信息 | 散射手电筒 |
| 精准 Prompt | 激活特定知识 | 聚焦手电筒 |
| 结构化 Prompt | 激活模式匹配 | 格式化输出 |
| 示例 Prompt | 激活类比推理 | Few-shot 学习 |

## Day 65: RAG 原理

**一句话**：通过向量检索注入外部知识，让模型"记住"训练后产生的信息。

### 核心流程

<div class="formula-box">

```
1. 用户问题 → Embedding → 问题向量 q
2. 向量检索：在数据库中找最相似的文档
   相似度 = q·d / (||q|| × ||d||)
3. 取 Top-K 相关文档片段
4. 注入 Prompt：
   "根据以下文档回答问题：
   [文档片段 1]
   [文档片段 2]
   问题：{用户问题}"
5. LLM 生成回答
```

</div>

### 三种记忆对比

| 记忆类型 | 更新频率 | 知识来源 | 局限性 |
|---------|---------|---------|--------|
| 参数记忆 | 训练时 | 预训练数据 | 静态、截止训练日期 |
| 上下文记忆 | 推理时 | Prompt | 受限于窗口长度 |
| RAG | 按需 | 外部数据库 | 依赖检索质量 |

## Day 66: 微调与 LoRA

**一句话**：通过梯度更新注入新知识，LoRA 用低秩适配器高效微调。

### 核心公式

<div class="formula-box">

```
全量微调：
W_new = W_old - η∂L/∂W
→ 更新所有参数
→ 可能覆盖旧知识

LoRA：
W_new = W_old + BA
B∈R^(d×r), A∈R^(r×k), r<<d
→ 只更新 A 和 B（低秩）
→ 保持 W_old 冻结
```

</div>

### 两种微调对比

| 方法 | 更新参数 | 灾难性遗忘 | 计算成本 | 可组合性 |
|------|---------|-----------|---------|---------|
| 全量微调 | 全部 | 严重 | 高 | 否 |
| LoRA | 低秩适配器 | 轻微 | 低 | 是 |

## Day 67: 灾难性遗忘

**一句话**：学习新知识时覆盖旧知识，是持续学习的核心挑战。

### 解决方案

| 方法 | 原理 | 效果 |
|------|------|------|
| **经验回放** | 混合新旧数据训练 | ⭐⭐⭐ |
| **弹性权重巩固** | 对重要权重施加惩罚 | ⭐⭐ |
| **LoRA** | 冻结原权重，只更新适配器 | ⭐⭐⭐ |
| **知识蒸馏** | 用旧模型指导新模型 | ⭐⭐ |

### 关键公式

<div class="formula-box">

```
EWC (弹性权重巩固)：
L = L_new + λΣᵢ Fᵢ(Wᵢ - Wᵢ_old)²
Fᵢ：Fisher 信息矩阵，度量权重ᵢ对旧任务的重要性
```

</div>

## Day 68: 少样本学习

**一句话**：通过少量示例让模型学会任务模式，无需微调。

### 三种学习模式对比

| 模式 | 示例数 | 训练 | 效果 |
|------|--------|------|------|
| Zero-Shot | 0 | 否 | 依赖预训练 |
| One-Shot | 1 | 否 | 临时学习 |
| Few-Shot | 2-10 | 否 | 强化模式 |
| Fine-tuning | 100+ | 是 | 最佳 |

---

## Day 69: 链式思维

**一句话**：要求模型"逐步推理"，激活推理链模式，减少跳步错误。

### 两种 Prompt 对比

| Prompt 类型 | 示例 | 准确率 |
|------------|------|--------|
| 直接回答 | "鸡兔同笼...各多少？" | 50% |
| CoT | "请逐步推理..." | 80%+ |

### 关键公式

<div class="formula-box">

```
CoT 生成：
P(answer | question) = Πₜ P(stepₜ | question, step₁...stepₜ₋₁)

每个步骤都基于前面所有步骤
→ 减少累积误差
```

</div>

**一句话**：综合运用 LLM 原理，设计高效 Prompt。

### Prompt 设计框架

<div class="formula-box">

```
优质 Prompt 框架：
1. 角色设定："作为 XX 专家..."
2. 任务说明："请完成 XX 任务..."
3. 上下文/示例："例如..."
4. 约束条件："要求：..."
5. 输出格式："请按以下格式..."
6. 推理要求："请逐步推理..."
```

</div>

<div class="formula-box">

```
好 Prompt:

3. 提出优化建议

代码：
[代码内容]

请逐步分析每个问题。"
```
