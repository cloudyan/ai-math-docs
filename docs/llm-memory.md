## 1. 核心问题

### 1.1 大模型如何"记住"知识？

<div class="formula-box">

```
传统认知：模型"记住"了知识
          ↓
          存储 → 检索 → 输出

实际机制：模型学会了预测模式
          ↓
          编码 → 激活 → 生成
```

</div>

### 1.2 三种记忆形式

| 记忆类型 | 存储位置 | 更新方式 | 生效时机 |
|---------|---------|---------|---------|
| **参数记忆** | 权重矩阵 W | 反向传播 | 每次前向传播 |
| **上下文记忆** | Attention 权重 | Prompt 注入 | 推理时即时 |
| **外部记忆** | 向量数据库 | 检索注入 | 推理时动态 |

## 2. 参数记忆：编码在权重矩阵中

### 2.1 什么是参数记忆

<div class="formula-box">

```
参数记忆 = 预训练过程中学到的知识
          编码在神经网络的权重矩阵中
          通过反向传播形成
          静态、截止训练日期
```

</div>

### 2.2 知识如何编码在权重中

<div class="formula-box">

```
训练过程：
1. 输入："北京是中国的首都"
        ↓
2. 模型预测：P(下一个词 | 上文)
        ↓
3. 计算损失：L = -log P("首都")
        ↓
4. 反向传播：∂L/∂W
        ↓
5. 更新权重：W = W - η∂L/∂W
        ↓
6. 重复数十亿次
        ↓
7. 知识被"雕刻"在权重矩阵中
```

</div>

### 2.3 权重矩阵的几何解释

<div class="formula-box">

```
知识不是存储在某个具体位置，
而是分布在整个权重矩阵的几何结构中！

"北京是中国首都"这个知识：
- 不是：{"北京": "中国首都"}
- 而是：权重矩阵中"北京"、"中国"、"首都"
        三个向量的几何关系
```

</div>

### 2.4 参数记忆的特点

| 特点 | 说明 | 影响 |
|------|------|------|
| **静态** | 训练后固定 | 无法学习新知识 |
| **分布式** | 知识分散在权重中 | 无法精确定位 |
| **概率性** | 存储的是模式 | 可能产生幻觉 |
| **高效** | 前向传播即可 | 推理速度快 |

## 3. 上下文记忆：编码在 Attention 权重中

### 3.1 什么是上下文记忆

<div class="formula-box">

```
上下文记忆 = 通过 Prompt 临时注入的信息
            编码在 Attention 权重中
            无需更新模型参数
            即时生效、仅在当前对话有效
```

</div>

### 3.2 Attention 机制详解

<div class="formula-box">

```
输入序列：[x₁, x₂, ..., xₙ]
        ↓
计算 Q, K, V：
Qᵢ = W_Q · xᵢ
Kⱼ = W_K · xⱼ
Vⱼ = W_V · xⱼ
        ↓
计算注意力分数：
Scoreᵢⱼ = Qᵢ · Kⱼ / √d
        ↓
Softmax 归一化：
Attentionᵢⱼ = softmax(Scoreᵢⱼ)
        ↓
聚合 Value：
Outputᵢ = Σⱼ Attentionᵢⱼ · Vⱼ
```

</div>

### 3.3 Prompt 如何"点亮"知识

<div class="formula-box">

```
Prompt: "作为中国首都，北京..."
        ↓
1. "首都"的 Query 向量 Q_首都
        ↓
2. 与所有 Key 计算相似度：
   Q_首都 · K_北京 = 高分 ✓
   Q_首都 · K_中国 = 高分 ✓
   Q_首都 · K_烤鸭 = 低分 ✗
        ↓
3. Softmax 放大差异：
   Attention_首都，北京 = 0.9
   Attention_首都，中国 = 0.08
   Attention_首都，烤鸭 = 0.02
        ↓
4. 聚合 Value：
   Output_首都 = 0.9 × V_北京 + 0.08 × V_中国
        ↓
5. "首都"的表示中主要包含"北京"、"中国"的信息
   其他无关信息被抑制
```

</div>

### 3.4 为什么能"点亮"

<div class="formula-box">

```
根本原因：

1. 预训练时，"首都"和"北京"经常共现
   → W_Q, W_K 学习了这种关联
   → Q_首都 和 K_北京 天然相似

2. Attention 机制是"相关性筛选器"
   → 高相关的 Key 获得高 Attention
   → 相关信息被聚合到当前表示

3. Softmax 放大效应
   → 微小差异被放大
   → 相关知识主导输出
```

</div>

## 4. 外部记忆：编码在检索结果中

### 4.1 什么是外部记忆

<div class="formula-box">

```
外部记忆 = 从向量数据库检索的知识
          通过 Prompt 注入模型
          动态、可更新
          弥补参数记忆的局限
```

</div>

### 4.2 RAG 工作流程

<div class="formula-box">

```
1. 用户问题："2025 年诺贝尔奖得主是谁？"
        ↓
2. 问题 Embedding：q = Embed(问题)
        ↓
3. 向量检索：
   相似度 = q · d / (||q|| × ||d||)
        ↓
4. 取 Top-K 相关文档
        ↓
5. 注入 Prompt：
   "根据以下文档回答问题：
   [文档 1 内容]
   问题：{用户问题}"
        ↓
6. 模型生成回答
```

</div>

### 4.3 外部记忆如何影响模型

<div class="formula-box">

```
1. 提供新信息
   模型参数中没有 2025 年的数据
   → 检索结果注入新信息

2. 约束生成
   "根据以下文档"指令
   → 模型倾向于从文档中提取答案

3. Attention 机制
   文档内容的 Key 与问题的 Query 匹配
   → 高 Attention 权重
   → 文档信息主导输出
```

</div>

## 5. 三种方式的根本差异

### 5.1 存储位置对比

<div class="formula-box">

```
┌─────────────────┬──────────────────┬─────────────────┐
│   参数记忆      │   上下文记忆     │   外部记忆      │
├─────────────────┼──────────────────┼─────────────────┤
│ 权重矩阵 W      │ Attention 权重   │ 向量数据库      │
│ W_Q, W_K, W_V   │ 动态计算         │ 文档 Embedding  │
│ 模型内部        │ 模型内部（临时） │ 模型外部        │
└─────────────────┴──────────────────┴─────────────────┘
```

</div>

### 5.2 更新机制对比

<div class="formula-box">

```
参数记忆：
训练 → 反向传播 → 更新 W → 知识改变
      ↓
   需要大量计算和数据
   通常以月/年为单位

上下文记忆：
新 Prompt → 重新计算 Attention → 知识改变
      ↓
   即时生效
   以秒为单位

外部记忆：
更新数据库 → 检索新文档 → 知识改变
      ↓
   分钟级更新
   以小时/天为单位
```

</div>

### 5.3 数学本质对比

| 维度 | 参数记忆 | 上下文记忆 | 外部记忆 |
|------|---------|-----------|---------|
| **数学对象** | 权重矩阵 W | Attention 矩阵 A | 向量集合{dᵢ} |
| **计算方式** | W·x（线性变换） | softmax(QKᵀ/√d)V | q·d（相似度） |
| **更新方式** | 梯度下降 | 重新计算 | 数据库更新 |
| **时间尺度** | 月/年 | 秒 | 小时/天 |
| **知识形式** | 隐式模式 | 显式文本 | 显式文档 |

## 6. Prompt 为什么能"点亮"知识

### 6.1 核心机制

<div class="formula-box">

```
Prompt 不是"检索"知识，
而是"激活"知识！

类比：
- 参数记忆 = 图书馆的藏书（静态存储）
- Prompt = 手电筒（照亮特定区域）
- Attention = 光线传播路径
```

</div>

### 6.2 向量空间解释

<div class="formula-box">

```
预训练后，向量空间形成：

"北京"向量附近：
- "中国"、"首都"、"人口"、"政治中心"...

Prompt: "作为中国首都，北京..."
        ↓
"首都"的 Query 向量
        ↓
与附近的 Key 向量高相似：
- Q_首都 · K_北京 = 高分
- Q_首都 · K_中国 = 高分
        ↓
Attention 聚合：
Output 主要包含"北京"、"中国"的 Value
        ↓
相关知识被"点亮"
```

</div>

### 6.3 概率分布解释

<div class="formula-box">

```
模型学习的是条件概率：
P(下一个词 | 上文)

Prompt 改变"上文"：
P(下一个词 | "北京") ≠ P(下一个词 | "作为中国首都，北京")

后者激活了不同的概率分布：
- "北京" → 可能接"有"、"是"、"的"
- "作为中国首都，北京" → 更可能接"政治中心"、"人口"
```

</div>

## 7. 应用场景选择

### 7.1 选择指南

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| **通用知识** | 参数记忆 | 高效、覆盖广 |
| **最新信息** | RAG | 可更新、准确 |
| **私有数据** | RAG | 安全、可控 |
| **临时指令** | 上下文记忆 | 灵活、即时 |
| **复杂推理** | 参数记忆 + CoT | 模型内在能力 |
| **事实问答** | RAG | 减少幻觉 |
| **创造性任务** | 参数记忆 | 概率生成是优势 |

### 7.2 组合使用

<div class="formula-box">

```
最佳实践 = 参数记忆 + 上下文记忆 + 外部记忆

例子：客服系统

参数记忆：
- 语言能力
- 推理能力
- 通用知识

上下文记忆：
- 当前对话历史
- 用户偏好
- 临时指令

外部记忆：
- 产品文档
- FAQ 知识库
- 最新政策
```

</div>

## 📚 总结

### 核心洞察

<div class="formula-box">

```
1. 大模型不是"记住"知识，而是学会预测模式

2. 参数记忆是"雕刻"在权重中的几何结构

3. 上下文记忆是 Attention 权重的动态模式

4. 外部记忆是通过检索注入的显式信息

5. Prompt 不是"检索"知识，而是"激活"知识

6. 三种方式各有优劣，最佳实践是组合使用
```

</div>

### 一句话总结

> **参数记忆 = 长期记忆（权重矩阵）**
> 
> **上下文记忆 = 短期记忆（Attention）**
> 
> **外部记忆 = 外部资料（向量检索）**
> 
> **Prompt = 注意力手电筒（点亮相关知识）**
