## 1. 核心问题

### 1.1 问题的本质

<div class="formula-box">

```
传统认知：模型"记住"了知识
          ↓
          存储 → 检索 → 输出

实际机制：模型学会了预测模式
          ↓
          编码 → 激活 → 生成
```

</div>

### 1.2 三种记忆形式对比

| 记忆类型 | 存储位置 | 更新方式 | 生效时机 |
|---------|---------|---------|---------|
| **参数记忆** | 权重矩阵 W | 反向传播 | 每次前向传播 |
| **上下文记忆** | Attention 权重 | Prompt 注入 | 推理时即时 |
| **外部记忆** | 向量数据库 | 检索注入 | 推理时动态 |

## 2. 参数记忆：编码在权重矩阵中

### 2.1 什么是参数记忆

<div class="formula-box">

```
参数记忆 = 预训练过程中学到的知识
          编码在神经网络的权重矩阵中
          通过反向传播形成
          静态、截止训练日期
```

</div>

### 2.2 知识如何编码在权重中

<div class="formula-box">

```
训练过程：
1. 输入文本："北京是中国的首都"
        ↓
2. 模型预测：P(下一个词 | 上文)
        ↓
3. 计算损失：L = -log P("首都")
        ↓
4. 反向传播：∂L/∂W
        ↓
5. 更新权重：W = W - η∂L/∂W
        ↓
6. 重复数十亿次
        ↓
7. 知识被"雕刻"在权重矩阵中
```

</div>

### 2.3 权重矩阵的几何解释

<div class="formula-box">

```
知识不是存储在某个具体位置，
而是分布在整个权重矩阵的几何结构中！

"北京是中国首都"这个知识：
- 不是：{"北京": "中国首都"}
- 而是：权重矩阵中"北京"、"中国"、"首都"
        三个向量的几何关系
```

</div>

### 2.4 特点与局限

| 特点 | 说明 | 影响 |
|------|------|------|
| **静态** | 训练后固定 | 无法学习新知识 |
| **分布式** | 知识分散在权重中 | 无法精确定位 |
| **概率性** | 存储的是模式 | 可能产生幻觉 |
| **高效** | 前向传播即可 | 推理速度快 |

## 3. 上下文记忆：编码在 Attention 权重中

### 3.1 什么是上下文记忆

<div class="formula-box">

```
上下文记忆 = 通过 Prompt 临时注入的信息
            编码在 Attention 权重中
            无需更新模型参数
            即时生效、仅在当前对话有效
```

</div>

### 3.2 Q-K-V 计算流程

<div class="formula-box">

```
输入序列：[x₁, x₂, ..., xₙ]
        ↓
计算 Q, K, V：
Qᵢ = W_Q · xᵢ
Kⱼ = W_K · xⱼ
Vⱼ = W_V · xⱼ
        ↓
计算注意力分数：
Scoreᵢⱼ = Qᵢ · Kⱼ / √d
        ↓
Softmax 归一化：
Attentionᵢⱼ = softmax(Scoreᵢⱼ)
        ↓
聚合 Value：
Outputᵢ = Σⱼ Attentionᵢⱼ · Vⱼ
```

</div>

### 3.3 Prompt 如何"点亮"知识

<div class="formula-box">

```
Prompt: "作为中国首都，北京..."
        ↓
1. "首都"的 Query 向量 Q_首都
        ↓
2. 与所有 Key 计算相似度：
   Q_首都 · K_北京 = 高分 ✓
   Q_首都 · K_中国 = 高分 ✓
   Q_首都 · K_烤鸭 = 低分 ✗
        ↓
3. Softmax 放大差异：
   Attention_首都，北京 = 0.9
   Attention_首都，中国 = 0.08
   Attention_首都，烤鸭 = 0.02
        ↓
4. "首都"的表示中主要包含"北京"、"中国"的信息
   其他无关信息被抑制
```

</div>

### 3.4 特点

| 特点 | 说明 | 影响 |
|------|------|------|
| **临时** | 仅在当前对话有效 | 对话结束即消失 |
| **动态** | 随 Prompt 变化 | 灵活可控 |
| **即时** | 无需训练 | 立即可用 |
| **有限** | 受上下文窗口限制 | 通常 4K-128K tokens |

## 4. 外部记忆：编码在检索结果中

### 4.1 什么是外部记忆

<div class="formula-box">

```
外部记忆 = 从向量数据库检索的知识
          通过 Prompt 注入模型
          动态、可更新
          弥补参数记忆的局限
```

</div>

### 4.2 RAG 工作流程

<div class="formula-box">

```
1. 用户问题："2025 年诺贝尔奖得主是谁？"
        ↓
2. 问题 Embedding：q = Embed(问题)
        ↓
3. 向量检索：
   相似度 = q · d / (||q|| × ||d||)
        ↓
4. 取 Top-K 相关文档
        ↓
5. 注入 Prompt
        ↓
6. 模型生成回答
```

</div>

### 4.3 特点

| 特点 | 说明 | 影响 |
|------|------|------|
| **动态** | 可随时更新 | 知识最新 |
| **精确** | 检索具体文档 | 减少幻觉 |
| **可追溯** | 知道信息来源 | 可验证 |
| **依赖检索** | 检索质量决定效果 | 需要好的检索系统 |

## 5. 根本差异对比

### 5.1 存储位置对比

<div class="formula-box">

```
┌─────────────────┬──────────────────┬─────────────────┐
│   参数记忆      │   上下文记忆     │   外部记忆      │
├─────────────────┼──────────────────┼─────────────────┤
│ 权重矩阵 W      │ Attention 权重   │ 向量数据库      │
│ W_Q, W_K, W_V   │ 动态计算         │ 文档 Embedding  │
│ 模型内部        │ 模型内部（临时） │ 模型外部        │
└─────────────────┴──────────────────┴─────────────────┘
```

</div>

### 5.2 数学本质对比

| 维度 | 参数记忆 | 上下文记忆 | 外部记忆 |
|------|---------|-----------|---------|
| **数学对象** | 权重矩阵 W | Attention 矩阵 A | 向量集合{dᵢ} |
| **计算方式** | W·x（线性变换） | softmax(QKᵀ/√d)V | q·d（相似度） |
| **更新方式** | 梯度下降 | 重新计算 | 数据库更新 |
| **时间尺度** | 月/年 | 秒 | 小时/天 |
| **知识形式** | 隐式模式 | 显式文本 | 显式文档 |

### 5.3 统一视角

<div class="formula-box">

```
三种记忆形式，本质上都是：
在向量空间中找到相关信息

参数记忆：
- 通过 W·x 变换
- 知识编码在 W 的几何结构中

上下文记忆：
- 通过 Q·K 匹配
- 知识激活在 Attention 模式中

外部记忆：
- 通过 q·d 检索
- 知识注入在 Prompt 中
```

</div>

## 6. Prompt 为什么能"点亮"知识

### 6.1 核心机制

<div class="formula-box">

```
Prompt 不是"检索"知识，
而是"激活"知识！

类比：
- 参数记忆 = 图书馆的藏书（静态存储）
- Prompt = 手电筒（照亮特定区域）
- Attention = 光线传播路径
```

</div>

### 6.2 向量空间解释

<div class="formula-box">

```
预训练后，向量空间形成：

"北京"向量附近：
- "中国"、"首都"、"人口"、"政治中心"...

Prompt: "作为中国首都，北京..."
        ↓
"首都"的 Query 向量
        ↓
与附近的 Key 向量高相似：
- Q_首都 · K_北京 = 高分
- Q_首都 · K_中国 = 高分
        ↓
Attention 聚合：
Output 主要包含"北京"、"中国"的 Value
        ↓
相关知识被"点亮"
```

</div>

### 6.3 为什么精准 Prompt 效果更好

<div class="formula-box">

```
模糊 Prompt:
"说一下北京"
→ 激活所有与"北京"相关的知识
→ 概率分布分散
→ 输出模糊

精准 Prompt:
"作为中国首都，北京在政治职能上有哪些特点？"
→ 精准激活"政治职能"相关知识
→ 概率分布集中
→ 输出精准
```

</div>

## 7. 应用场景选择

### 7.1 选择指南

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| **通用知识** | 参数记忆 | 高效、覆盖广 |
| **最新信息** | RAG | 可更新、准确 |
| **私有数据** | RAG | 安全、可控 |
| **临时指令** | 上下文记忆 | 灵活、即时 |
| **复杂推理** | 参数记忆 + CoT | 模型内在能力 |
| **事实问答** | RAG | 减少幻觉 |
| **创造性任务** | 参数记忆 | 概率生成是优势 |

### 7.2 组合使用

<div class="formula-box">

```
最佳实践 = 参数记忆 + 上下文记忆 + 外部记忆

例子：客服系统

参数记忆：
- 语言能力
- 推理能力
- 通用知识

上下文记忆：
- 当前对话历史
- 用户偏好
- 临时指令

外部记忆：
- 产品文档
- FAQ 知识库
- 最新政策
```

</div>
